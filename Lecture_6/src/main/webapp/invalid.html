Hendelse invalid

FORM submit blir automatisk kansellert når FORM kontroll-elementer har ugyldige data, og hendelse invalid vil da skje (fyre) på kontroll-element med ugyldige data.

Eksempler på FORM kontroll-elementer er f.eks. input, select og radio.

Data er ugyldig hvis FORM kontroll-element attributter som pattern og required ikke er oppfylt.

Hendelse invalid kan også bli trigget uten først en FORM submit. Det skal vi se på i faget DAT152 der vi også skal ta kontroll på valideringsvinduet når data er ugyldig. Da skal vi også se hvordan vi med JavaScript kan sette tiltand til invalid, f.eks. hvis felt for passord og felt for repetert passord har ulike verdier.

Visining av data som ugyldig gjøres her ved CSS sin pseudoklasse invalid.

Callback for hendelse invalid er lagt til ved følgende JavaScript for hvert FORM kontroll-element:

refInputElement.addEventListener("invalid",callback);

Dette dokumenet inneholder følgende HTML:

<form method="post" action="action.html">
    <fieldset>
        <label for="fornavn">Fornavn:</label>
        <input type="text" id="fornavn" name="fornavn" placeholder="Dine fornavn"
            title="Fornavn må starte med stor bokstav, så små bokstaver"
            required pattern="\s*\p{Lu}\p{Ll}+(\s+\p{Lu}\p{Ll}+)*\s*"/>

        <label for="etternavn">Etternavn:</label>
        <input type="text" id="etternavn" name="etternavn" placeholder="Ditt etternavn"
            title="Etternavn må starte med stor bokstav, så små bokstaver"
            required pattern="\s*\p{Lu}\p{Ll}+\s*"/>
 
        <label for="telefon">Telefon:</label>
        <input type="tel" id="telefon" name="telefon" placeholder="Telefonnummer"
            title="Telefonnummer må inneholde minst 8 siffre, og kun siffre"
            required pattern="\s*\+?(\s*\d){8,}\s*"/>

        <button type="submit">Send data</button>
    </fieldset>
</form>
